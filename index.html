Claude Code ユースケース紹介

COREWOでの活用事例 — 「集めて、まとめる」を自動化する

ファイナンス支援は情報が命。決算書、kintone、MF クラウド会計、補助金ポータル… 複数のシステムに散らばった情報を1つの資料にまとめる作業は、手作業では数時間かかります。

COREWOでは、この「集めて→まとめる」をClaude Codeに任せ、 人間は「読んで→判断する→提案する」に集中しています。

未経験メンバーが多いチームだからこそ、情報収集・資料作成の質をAIで底上げしつつ、 考える部分は人がやる。AIとの共存で、ファイナンス支援の現場を変えています。

ユースケース① 決算前MTGブリーフィング

顧客との決算前ミーティングに向けて、企業情報・財務データ・市場分析・制度情報をまとめた包括的なブリーフィングを自動生成します。

実行フロー

Step 1. 顧客情報の取得 kintone
会社名でkintoneを検索し、顧客ID・担当者情報・連絡窓口を取得
Step 2. 財務データの収集 MFクラウド会計 PDF
MFクラウド会計からBS/PL/試算表を取得。決算書PDFをClaude Visionで読み取り、 別表一（本店住所・納税地）、別表二（株主情報・同族判定）、概況書（設立年月日）を構造化
Step 3. 外部情報の調査 Web調査
会社HP・プレスリリース・採用情報から事業内容と強みを分析。 市場規模・PEST分析・5F分析を実施。関連する補助金・融資制度の最新情報を調査
Step 4. ブリーフィング生成 Notion
テンプレートに沿って8セクション構成のブリーフィングを生成。 Notion顧客DBに子ページとして追加し、財務プロパティ（売上高・営業利益・資産合計等）を更新
Step 5. 完了報告 Slack
作成先のNotionページURL、更新したプロパティの概要をSlack DMで報告

ユースケース② 補助金情報の自動収集

毎朝9時に6つの情報ソースから補助金・助成金の最新情報を並列収集し、 Notionデータベースに蓄積。変更があればSlackに自動通知します。

収集ソース（6種）

jGrants
デジタル庁の補助金API。受付中の補助金を一括取得し詳細情報を収集
持続化補助金
小規模事業者持続化補助金の公募状況・締切・採択発表を追跡
ものづくり補助金
回次ごとのスケジュール（23次まで）と採択結果を追跡
新事業進出補助金
公募スケジュールと申請期限を自動取得
省力化投資補助金
カタログ注文型・一般型の公募状況を監視
東京都中小企業振興公社
躍進的事業推進、DX活用、環境、新製品開発、創業助成の5事業を追跡

MCP連携 = Claude Codeの手足

MCP（Model Context Protocol）により、Claude Codeから外部サービスのAPIを直接呼び出せます。

サービスできること活用例
kintone レコード検索・取得、フォーム管理、アプリ操作 顧客情報・担当者の取得、顧客IDの特定
Notion ページ作成・更新、DB検索、コメント ブリーフィング出力、補助金DB更新、顧客マスター管理
Slack メッセージ送受信、チャンネル操作、スレッド返信 完了報告、補助金変更通知、チーム連絡
MFクラウド会計 仕訳登録、試算表取得、BS/PL取得、取引先管理 決算データの自動取得、財務分析用データ収集
Google Drive ファイル検索・取得 決算書PDF等の参照
データの流れ: 自然言語で指示 → Claude Code がMCP経由でAPI実行 → 結果を構造化して出力（Notion / Slack / ローカルファイル）

Hook = 安全装置とガードレール

AIが自律的に動く中でも、事故を防ぐためのHookを設定しています。

タイミングHookやること
PreToolUse .env ステージング防止 git add .env や git add . を検知して自動ブロック。シークレットの漏洩を防止
PreToolUse GDrive トークンリフレッシュ Google Drive系ツール呼び出し前にOAuthトークンを自動更新。認証切れによるエラーを防止

パーミッション設定

書き込み禁止ファイル（deny）:
.env / .env.* / credentials.* / secrets.* / .git/**

これらのファイルへのEdit/Writeは、Claude Codeがどのような指示を受けても実行できません。

ナレッジ蓄積 = チームの知的資産をストックする

日々の業務で得た知見、上司のノウハウ、調査結果をナレッジファイルとして蓄積し、 次の業務に活かせる形で管理しています。

3つのナレッジサイクル

/distill — 上司の知見を資産化する
上司（藤田さん）との会話や指摘をFleeting Notes（走り書き）として記録。 /distill コマンドでPermanent Note（恒久的なナレッジ）に昇華。 属人的なノウハウを、チームで参照できる形に変換します。
/dream — メモリの整理・統合
Claude Codeのメモリファイル（セッション間で持続する記憶）を定期的に整理。 重複の排除、古い情報の更新、関連情報の統合を自動で実行します。
/review-knowledge — 品質チェック
蓄積したナレッジファイルの鮮度・正確性をレビュー。 古くなった情報のフラグ付け、出典切れの検出を行います。
